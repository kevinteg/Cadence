Build Cadence v1 set out on April 15, 2026 as a proving-ground project with a concrete external forcing function: have a cognitive operating system working from day one at a new role. What it became was something narrower and more honest. The original architecture — Sessions, Markers, multiple agent modes, a per-agent voice register, DoD checklists as the project format — gave way incrementally to a design that trusted the project file itself as the durable record. The three most consequential structural pivots were: the voice collapse (one agent, one register set by the invoked verb, documented in v3-voice-and-contracts), the removal of the Session/Marker concept (define-checkpoint-and-restore was proposed and then immediately superseded by remove-session-concept, which recognized that pursuit/project/action is already a write-ahead log of intent), and the migration from Definitions of Done to Intent-and-Actions (narrative motivation first, atomic actions second, completion judged against felt-sense of done rather than a checklist sweep). The plugin-only migration consolidated what had been split between local .claude/commands/ and a plugin directory, making the distributable artifact the single source of truth.

The durable artifacts that shipped are these: a bundled TypeScript CLI at cadence-plugin/bin/cadence that covers both read (status, scan, project-activity, ideas, captures) and write (create-project, check, set-status, add-waiting-for, write-capture) surfaces, removing ad-hoc file mutations from skills entirely; a twelve-skill verb catalog covering brainstorm, develop, promote, start, complete, cancel, waiting, narrate, reflect, capture, close, and reconcile; the plugin manifest and plugin.json packaging that allows installation via claude --plugin-dir; two dedicated subagents — cadence:narrator and cadence:reconciler — that handle bulk data ingestion in isolation to keep the main conversation context clean; and the watermark-resume narrative system, where each generated narrative carries frontmatter (consumed_from_commit, consumed_through_commit, cadence, projects_consulted) that doubles as the read pointer into the git stream for the next run of the same cadence. The provocation deck shipped as 58 cards across four categories. The Ideas entity and the Wandering pursuit gave unattached seeds a home outside any finite pursuit. WIP guardrails and the reconciler formalized the system's self-maintenance surface.

Six projects were dropped. hook-handles-uninit-repos became redundant when plugin-session-start-hook shipped clean uninitialized-repo handling at the CLI layer. test-personal-repo was scoped out — plugin validation happened in-repo through per-project user-story checks, and the "use Cadence in a separate personal repo" work belongs to its own pursuit. define-checkpoint-and-restore proposed a richer Session schema and then immediately ran into remove-session-concept, which made the entire Session vocabulary unnecessary; the dropped reason is precise: "superseded." promote-state-verbs-to-commands proposed converting five state-modifying verbs to deterministic command templates, but the combination of TRIGGER/SKIP discipline from enforce-explicit-verb-skill-triggers, bundled CLI writes, and active-session state changes dissolved the original premise — walking each candidate revealed only /capture would genuinely fit, and that conversion was low-leverage. weekly-review-nudge had its scope absorbed into surface-recent-activity-in-splash, which added the late-week LP preview as a rule inside nextSteps() rather than a separate project. cross-verb-numbered-selection was the most recently dropped project: created and dropped April 29, the judgment was that /status N is sufficient coverage and fuzzy ID matching handles the rest; it may resurface if real use reveals pain.

One idea completed the full pipeline during this pursuit's lifetime. start-complete-verbs was seeded in the wandering pursuit on April 21, developed the same day, and promoted directly to project:implement-lifecycle-verbs — which shipped the explicit verb lifecycle (start, complete, cancel) and upward completion, replacing the overloaded "do" verb. That single idea-to-project arc represents the pipeline working as designed. At closure, no unresolved ideas exist under build-cadence-v1. One seed now lives under make-cadence-public: the claude-code-marketplace-listing idea, deferred from the reduce-install-friction project's scope.

What's next is visible in the someday pursuit that has been accumulating scope throughout this work. make-cadence-public holds the questions the build pursuit deliberately parked: the marketplace listing, the install-friction reduction work, whether the plugin repository should separate from the product development repository, and how to version and distribute a v1 release. Two other frontiers emerged but were not addressed: performance and indexing (the CLI's full-repo scan works fine at this repo's scale but is unvalidated against larger corpora, and there is no incremental index), and the first real external user (the personal-repo test that was scoped out of this pursuit). The narrative system has daily and weekly contracts defined and journeyed; monthly and annual contracts are stubbed as generic McAdams and will need their own cadence-specific shapes when sustained use generates enough history to make them meaningful. The system is ready to be used. The build pursuit is done.
